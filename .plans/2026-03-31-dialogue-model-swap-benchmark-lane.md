## Goal

Run the next narrow dialogue benchmark experiment by swapping the dialogue chunk-transcription model in the YAML config to a stronger single-model path, then measure whether that improves score versus the current human-gold baseline.

---

## Overview

This plan continues the active dialogue benchmark lane documented in `.plans/2026-03-30-dialogue-benchmark-iteration-against-human-gold.md`. That prior session established a durable baseline of **42.44% accuracy / 92.01% coverage** against the human-reviewed `cod-test` dialogue benchmark, then stopped after one blocker-reveal and one scored regression. The explicit next-step hypothesis from that handoff was **Option B**: test whether the current flash-lite-first heterogeneous chunk-transcription path is itself driving transcript drift, speaker collapse, and missing tail coverage.

The narrow config surface for this lane is `configs/cod-test-dialogue-benchmark-baseline.yaml`. Today’s experiment should stay inside that narrow lane and avoid product overreach. The expected main change is to swap the dialogue chunk-transcription target from the current lightweight primary (`google/gemini-3.1-flash-lite-preview`, `thinking: low`) to a stronger single-model candidate with higher reasoning budget, while keeping the prompt generic and avoiding any fixture-specific hints. We want an honest benchmark answer to a simple question: does a stronger evaluator/transcriber materially improve relative score against the new human gold benchmark?

The result must be judged strictly against the existing baseline and written back into both the markdown narrative and the durable benchmark ledger. If the stronger-model swap clearly improves score, it can become the new active baseline candidate or feed a follow-up lane. If it regresses or proves too noisy, we should stop cleanly and record that the model-quality theory was not sufficient on its own.
